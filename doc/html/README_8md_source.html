2 LibreDWG
3 ---------
4 LibreDWG is a free C library to read and write DWG files. This program is part of the GNU project, released under the aegis of GNU. It is licensed under the terms of the GNU General Public License version 3 (or at you option any later version).
5 
6 DWG
7 ----
8 DWG is a file format created in the 70's for the emerging CAD applications. Currently it is the native file format of AutoCAD, a proprietary CAD program developed by AutoDesk.
9 
10 LibreDWG is a fork from LibDWG due to its usage of Esperanto, which we think is not the best strategy for a free software project which aims to get lots of contributors. LibreDWG is written in English. Since we forked LibDWG (May 16, 2009), its development seems to have been inactive. At the moment (September 25, 2009) our decoder is almost done. We are reaching 100% read support in our data structures parser but we still need a test suite. Among the example applications we wrote using LibreDWG is an initial SVG conversion tool. Please contact us if you can help.
11 
12 The program dwg_ps (in examples/) reads DWG and outputs Postscript; it depends on pslib: <http://pslib.sourceforge.net>. In many GNU/Linux distributions, this is available by installing the pslib-dev package.
13 If the configure script does not find pslib, dwg_ps will not be built.

26 Building and Installing the Library
27 -----------------------------------
28 
29 See INSTALL for generic instructions. Basically, you do:
30 
31  $ ./autogen.sh (if you checked out the source code from git)
32  $ ./configure [options]
33  $ make
34  $ make check # optional but strongly encouraged while LibreDWG is alpha
35  $ make install
36 
37 
38 This builds and installs various files in the "installation dirs":
39 
40 * $libdir ($exec_prefix/lib) -- library files (libredwg.*)
41 
42 * $includedir ($prefix/include) -- dwg.h
43 
44 * $infodir ($datarootdir/info) -- LibreDWG.info
45 
46 Customisation
47 --------------
48 Use "./configure --help" to see a full list of options you can use to
49 customize the installation dirs. Other options recognized by configure are:
50 
51 * --enable-trace
52 
53 NB: This is an experimental feature that is not in its final form.
54 
55 Enable runtime tracing (default: no). When enabled, the environment variable LIBREDWG_TRACE is consulted on the first decode/encode attempt. Its value is an integer: 0 (no output) through 9 (full verbosity).
56 
57 * --enable-write
58 
59 NB: This is an experimental feature that is not in its under construction, so its highly unstable.
60 
61 Enable write support (default: no). When enabled you activate the write support for the dwg format.

63 Example usage of the library
64 -----------------------------
65 
66 The examples directory contains some application code that uses our lib:
67 
68 * load_dwg.c -- This is a skeleton code. Look there if you want to have a
69  quickly glance on how to use the library.
70 
71 * dwg_ps.c -- Opens a DWG file and outputs an PostScript file. This code
72  depends on the PSlib library. On Debian-based systems this is usually
73  available as a package named pslib-dev
74 
75 * testSVG.c -- Opens a DWG file and outputs an SVG (Scalable Vector Graphics)
76  file. SVG is a W3C standard for 2d vector graphics. You can open these
77  files on several free software tools. We recomend Inkscape, a free software
78  vector graphics editor. The testSVG.c program will not handle 3d content
79  from DWG since SVG only supports 2-dimensional images.
80 
81 * get_bmp.c -- Extracts the bmp thumbnail of a dwg file when it is available.
82 
83 * rewrite.c -- Load, rewrite and try to re-read a existing dwg file. Just for writing test. Still failing (as in 08/2010)
